## Windows 构建踩坑记录

### 坑 1：macOS `._*` 元数据文件污染

**现象**：
```
Error: Patch file found for package claude-agent-sdk which is not present at node_modules/._@anthropic-ai/claude-agent-sdk
[apply-openclaw-patches] Patch does not apply cleanly: ._openclaw-gateway-entry.patch
```

**原因**：从 Mac 往 exFAT 格式 U 盘拷文件时，macOS 自动生成 `._*` 隐藏元数据文件（resource fork）。这些文件被 patch-package 和 openclaw patch 脚本当成真实文件处理，导致报错。

**解决**：在 Mac 上挂载 U 盘后批量删除：
```bash
find /Volumes/UCLAW/LobsterAI -name "._*" -delete
```

**预防**：每次从 Mac 往 U 盘拷代码前先清一遍，或改用 `rsync`（默认不写入 `._` 文件）。

---

### 坑 2：`npm install` 被杀毒软件 EPERM 中断

**现象**：
```
EPERM: operation not permitted, rmdir 'D:\LobsterAI\node_modules\cytoscape'
error signal SIGINT
```

**原因**：Windows 杀毒软件实时扫描占用了目录，npm 清理旧 node_modules 时无法删除。

**解决**：将项目目录加入杀毒白名单，或重启后再删 node_modules 重装。

---

### 坑 3：缺少 `pnpm`

**现象**：
```
Missing required command: pnpm
```

**原因**：OpenClaw 构建依赖 `pnpm`，Windows 默认不带。

**解决**：
```bat
npm install -g pnpm
```

---

### 坑 4：git 拒绝操作 exFAT 目录

**现象**：
```
fatal: detected dubious ownership in repository at 'D:/openclaw'
'D:/openclaw' is on a file system that does not record ownership
```

**原因**：openclaw 被克隆到 U 盘（exFAT 文件系统），exFAT 不记录文件所有权，git 出于安全拒绝操作。

**解决方案对比**：
- 方案 A：`git config --global --add safe.directory D:/openclaw`（治标）
- 方案 B：把整个项目拷到系统盘（NTFS）再构建 ✅（推荐）

---

### 坑 5：PowerShell 的 `set` 不设置环境变量

**现象**：执行 `set OPENCLAW_SKIP_ENSURE=1` 后变量未生效，openclaw 流程仍然触发。

**原因**：`PS>` 提示符表示是 PowerShell 环境，PowerShell 里 `set` 是 `Set-Variable` 的别名，设置的是 PowerShell 变量而非系统环境变量，子进程（npm）读不到。

**正确写法**：
```powershell
$env:OPENCLAW_SKIP_ENSURE = "1"
npm run dist:portable-win
```

---

### 坑 6：`predist:portable-win` 强制触发 OpenClaw 构建

**现象**：`OPENCLAW_SKIP_ENSURE` 只跳过版本切换，不跳过整个编译流程，`predist:portable-win` 仍会执行完整的 `openclaw:runtime:win-x64`。

**根本原因**：我们写的 `dist:portable-win` 脚本绑了 `predist:portable-win`，而后者调用了 openclaw 构建链。

**最终决策**：便携版不需要 OpenClaw（目标用户用内置 yd_cowork 引擎完全够），将项目拷到系统盘构建，openclaw 会正常克隆到 `C:\openclaw`（NTFS），git 不报错，整个流程跑通。
